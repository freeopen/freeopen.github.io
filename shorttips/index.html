避坑指南

  • StrongSwan 不支持 ECC 证书，支持 RSA 证书，而 acme.sh 默认生成 ECC 证书， 须手动指定成 RSA 才能兼容。 -- 2023.11.2

  • gogs ssh 连接时，当客户端机器同时存在RSA、ECC 等多版本证书时，只能自动连接一个，如果自动连接的证书不是你正在配置的那个，永远连接失效，没配置的情况下，看不到错误日志输出，只能盲猜; 重要信息在 gogs/log/hooks/serv.log 中，注意在 app.ini 的日志配置部分选择 file 模式; 不要启用内置ssh服务，用服务器默认的 sshd 即可，记得把配置文件 sshd_config 指向到 /home/git/.ssh/authorized_keys, 否则 gogs 没有权限写入, 打开 PubkeyAuthentication yes 。 -- 2023.11.7

  • git clone 时，遇到 error: failed to encode 'src/System/ZSplash.ZC' from UTF-8 to cp437 时，新建 .git/info/attributes 文件， 增加 *.ZC text working-tree-encoding=UTF-8, 注意自己初始化目录并fetch。 -- 2023.11.9

  • google.com 搜索 soft_name 94FBR 可找到破解版或激活码。 -- 2023.11.30

  • 不同类型 cpu ( x86 / arm ) 的非对称密钥生成的证书是不兼容的，即使采用的是同样的公钥和私钥。 -- 2023.12.14
